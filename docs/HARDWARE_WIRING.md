# 🔌 مخطط توصيلات العتاد — Raspberry Pi 4

> يوثّق هذا الملف توصيل كل وحدة على أقطاب الراسبيري باي 4، بعد توليد منافذ UART
> إضافية عبر `dtoverlay`. الترقيم **BCM** للأقطاب (GPIOxx) ورقم **الطرف الفيزيائي**
> (Pin) على رأس الـ 40 طرف.

---

## ملخّص سريع — من يأخذ ماذا

| الوحدة | الناقل | منفذ النظام | قطب الراسبيري (الطرف الفيزيائي) | السرعة |
|--------|--------|-------------|--------------------------------|--------|
| سيرفوات يمين (PCA9685 #1 @0x40) | I2C | — | SDA=GPIO2(p3) · SCL=GPIO3(p5) | — |
| سيرفوات شمال (PCA9685 #2 @0x44) | I2C | — | SDA=GPIO2(p3) · SCL=GPIO3(p5) | — |
| قاطع تغذية السيرفوات (MOSFET) | GPIO | — | GPIO17 (p11) | — |
| **IMU — BNO085** | UART0 | `/dev/serial0` | IMU‑TX → **GPIO15 RX (p10)** | 115200 |
| **ليدار — TFmini/TF‑Luna** | UART3 | `/dev/ttyAMA2` | Lidar‑TX → **GPIO5 RX (p29)** | 115200 |
| **GPS** | UART4 | `/dev/ttyAMA3` | GPS‑TX → **GPIO9 RX (p21)** | 9600 |
| **كاميرا حرارية** | UART5 | `/dev/ttyAMA4` | Cam‑TX → **GPIO13 RX (p33)** | (يُحدَّد) |
| **رطوبة التربة** | USB‑Serial | `/dev/ttyUSB0` | منفذ USB (محوّل USB‑UART) | — |
| **حساس الغاز (DO رقمي)** | GPIO | — | DO → **GPIO6 (p31)** | — |
| **DHT22 (حرارة + رطوبة الجو)** | 1‑Wire رقمي | — | DATA → **GPIO16 (p36)** | — |

> ⚠️ **القاعدة الذهبية للـ UART:** خط **TX** للحساس يدخل دائماً على **RX** للراسبيري،
> وخط **RX** للحساس (إن وُجد) يدخل على **TX** للراسبيري. (تقاطع متصالب).

   POWER  GPIO17 (11)(12) GPIO18
         GPIO27 (13)(14) GND

## التفصيل لكل وحدة

### 1) IMU — BNO085 (UART0 / `/dev/serial0`) — موجود مسبقاً
| سلك الـ IMU | يذهب إلى |
|------------|----------|
| VIN/3V3 | 3V3 (p1 أو p17) |
| GND | GND |
| **TX (UART/RVC)** | **GPIO15 RX — الطرف 10** |
| RX (اختياري) | GPIO14 TX — الطرف 8 |
| PS0/PS1 | حسب وضع RVC (راجع لوحة الاستشعار) |

يبقى على UART0 بلا تغيير لأنه حسّاس للتوقيت (موازنة الجسم).

### 2) ليدار — TFmini / TF‑Luna (UART3 / `/dev/ttyAMA2`)
| سلك الليدار | يذهب إلى |
|------------|----------|
| موجب (+5V) | 5V (p2 أو p4) — TF‑Luna يقبل 3.3~5V، TFmini يحتاج 5V |
| سالب (GND) | GND |
| 3.3V (مرجع منطق) | 3V3 (p1) إن طلب الموديل ذلك، وإلا اتركه |
| **TX** | **GPIO5 RX — الطرف 29** |
| RX (اختياري) | GPIO4 TX — الطرف 7 |

الليدار يحتاج فقط أن **يُقرأ** منه، فيكفي توصيل TX→RX.

### 3) GPS (UART4 / `/dev/ttyAMA3`)
| سلك الـ GPS | يذهب إلى |
|------------|----------|
| VCC | 3V3 أو 5V حسب الموديل (NEO‑6M/7M = 3.3~5V) |
| GND | GND |
| **TX** | **GPIO9 RX — الطرف 21** |
| RX | GPIO8 TX — الطرف 24 (لإرسال أوامر الضبط، اختياري) |

### 4) كاميرا حرارية (UART5 / `/dev/ttyAMA4`)
| سلك الكاميرا | يذهب إلى |
|-------------|----------|
| VCC | حسب الموديل (3.3V أو 5V) |
| GND | GND |
| **TX** | **GPIO13 RX — الطرف 33** |
| RX | GPIO12 TX — الطرف 32 (إن احتاجت أوامر) |

⚠️ **تأكّد من منطق الكاميرا 3.3V**. إن كان خرجها 5V استخدم مقسّم جهد/مبدّل مستوى على خط TX.

### 5) حساس الغاز — خرج رقمي DO (GPIO6 / الطرف 31)
| سلك الحساس | يذهب إلى |
|-----------|----------|
| VCC | 5V (موديولات MQ تحتاج مسخّن 5V) |
| GND | GND |
| **DO** | **GPIO6 — الطرف 31** عبر **مقسّم جهد** (انظر التحذير) |
| AO | لا يُوصَّل (نستخدم الرقمي فقط) |

🚨 **تحذير مستوى الجهد:** موديولات MQ تُخرج DO عند جهد التغذية. لو غذّيتها 5V سيكون
DO = **5V** ويُتلف قطب الراسبيري (الحد 3.3V). الحل: مقسّم جهد بسيط على DO
(مثلاً 10kΩ + 20kΩ: من DO → 10k → نقطة القراءة → 20k → GND) أو مبدّل مستوى منطقي.

### 6) DHT22 — حرارة + رطوبة الجو (1‑Wire رقمي / GPIO16 / الطرف 36)
| سلك الـ DHT22 | يذهب إلى |
|--------------|----------|
| VCC (الطرف 1) | **3V3** (p1) — ليبقى خط البيانات عند 3.3V آمناً للراسبيري |
| DATA (الطرف 2) | **GPIO16 — الطرف 36** |
| (الطرف 3) | لا يُوصَّل (NC في DHT22) |
| GND (الطرف 4) | GND |

> 🔧 **مقاومة سحب إلزامية:** ضع **10kΩ بين DATA و VCC (3V3)**. أغلب موديولات DHT22
> (3 أطراف) فيها المقاومة مدمجة — تحقّق من لوحتك. DHT22 بروتوكول سلك-واحد خاص (ليس
> HIGH/LOW بسيط)، يُقرأ بمكتبة `adafruit_dht` ويعطي **حرارة °C + رطوبة %**.

### 7) رطوبة التربة — USB‑Serial (`/dev/ttyUSB0`)
| التوصيل | الملاحظة |
|---------|----------|
| الحساس → محوّل USB‑UART → منفذ USB بالراسبيري | يظهر كـ `/dev/ttyUSB0` |

> انتقل من ADS1115 (I2C) إلى منفذ USB مستقل — يحرّر ناقل I2C ولا يزاحم السيرفوات.
> ⚠️ إن وُصِّل أكثر من محوّل USB‑Serial فالاسم قد يصبح `ttyUSB1`؛ ثبّته عبر udev
> (`SUBSYSTEM=="tty", ATTRS{idVendor}=="...", SYMLINK+="spider_soil"`) أو تحقّق بـ `ls /dev/ttyUSB*`.

## تفعيل UART الإضافية — `/boot/firmware/config.txt`

> على إصدارات Raspberry Pi OS القديمة المسار `/boot/config.txt`.

```ini
# تفعيل الـ UART الأساسي (UART0 لـ IMU)
enable_uart=1

# توليد منافذ UART إضافية على الأقطاب
dtoverlay=uart3      # ليدار        → /dev/ttyAMA2  (GPIO4 TX p7 / GPIO5 RX p29)
dtoverlay=uart4      # GPS          → /dev/ttyAMA3  (GPIO8 TX p24 / GPIO9 RX p21)
dtoverlay=uart5      # كاميرا حرارية → /dev/ttyAMA4  (GPIO12 TX p32 / GPIO13 RX p33)
```

### خطوات ما بعد التعديل
1. **حرّر منفذ UART0 من الكونسول** حتى يبقى للـ IMU:
   ```bash
   sudo raspi-config
   # Interface Options → Serial Port
   #   "login shell over serial?" → No
   #   "serial port hardware enabled?" → Yes
   ```
   وتأكّد من إزالة `console=serial0,115200` من `/boot/firmware/cmdline.txt`.
2. **أعد التشغيل** ثم تحقّق من ظهور المنافذ:
   ```bash
   ls -l /dev/ttyAMA*
   # المتوقّع: ttyAMA0 (UART0) · ttyAMA2 (uart3) · ttyAMA3 (uart4) · ttyAMA4 (uart5)
   dmesg | grep -i tty
   ```
   ⚠️ **هام:** أسماء `ttyAMAx` قد تختلف حسب ترتيب الإقلاع. ثبّتها بأسماء مستقرّة عبر udev
   (انظر أدناه) أو عدّل المنافذ في الكود حسب ما يظهر فعلياً.
3. (موصى به) **أسماء ثابتة عبر udev** — أنشئ `/etc/udev/rules.d/99-spider-uart.rules`:
   ```
   # اربط كل UART بقطبه الفيزيائي لاسم ثابت
   SUBSYSTEM=="tty", KERNELS=="fe201000.serial", SYMLINK+="spider_imu"
   SUBSYSTEM=="tty", KERNELS=="fe201600.serial", SYMLINK+="spider_lidar"
   SUBSYSTEM=="tty", KERNELS=="fe201800.serial", SYMLINK+="spider_gps"
   SUBSYSTEM=="tty", KERNELS=="fe201a00.serial", SYMLINK+="spider_thermal"
   ```
   ثم استخدم `/dev/spider_lidar` … إلخ في الكود (عناوين الـ KERNELS تُؤخذ من `udevadm info`).

## تذكيرات كهربائية

- **لا تُغذِّ السيرفوات من الراسبيري إطلاقاً** — مصدر 5V/6V منفصل عالي التيار، والأرضي مشترك (GND مشترك).
- **حساس الغاز MQ يسخن ويسحب ~150mA** — يفضّل مصدر 5V خارجي مع GND مشترك، ولا تنسَ مقسّم الجهد على DO.
- **الأرضي المشترك (Common GND)** بين الراسبيري وكل مصادر التغذية شرط أساسي لعمل UART والمنطق الرقمي.
- **منطق 3.3V فقط على أقطاب الراسبيري** — أي خرج 5V يحتاج مبدّل مستوى أو مقسّم جهد.
- **DHT22:** غذِّه من 3V3 + مقاومة سحب 10kΩ على DATA؛ بروتوكول سلك-واحد (لا DO بسيط).
- ميزانية I2C: السيرفوات فقط (PCA9685×2) على الناقل — رطوبة التربة انتقلت لـ USB.
