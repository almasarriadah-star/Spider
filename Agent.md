# Spider Project — Agent Reference

## ملف التوازن (`ground_calibration_test.py`)

### الغرض
اختبار ومعايرة وضعية التوازن للأرجل الستة للروبوت العنكبوت (Hexapod) على الأرض.
بيحرّك كل السيرفوهات لوضعية التوازن المثالية عشان الروبوت يوقف بشكل مستقر.

### الهاردوير
- **Right PCA** — عنوان `0x40` → 9 قنوات (R0-R8)
- **Left PCA** — عنوان `0x44` → 9 قنوات (L0-L8)
- كل جانب = 3 أرجل × 3 مفاصل (coxa, femur, tibia)
- زاوية الحركة: `45° - 135°`

### بيانات المعايرة
كل قناة عندا زاوية توازن محددة:
- **RIGHT**: R0=90, R1=75, R2=92, R3=92, R4=65, R5=96, R6=93, R7=65, R8=94
- **LEFT**: L0=90, L1=57, L2=81, L3=85, L4=74, L5=95, L6=94, L7=66, L8=98

### الأوامر التفاعلية
- `calibrate` — تحريك سلس (smooth) لكل السيرفوهات لوضعية التوازن
- `direct` — تحريك مباشر فوري بدون سلاسة
- `status` — عرض زوايا التوازن الحالية
- `off` — فصل كل السيرفوهات (أمان)
- `exit` — فصل + خروج من البرنامج
- `help` — قائمة الأوامر

### ملاحظات مهمة
- التحريك السلس بيستخدم `MOVE_DELAY = 0.025` ثانية بين كل درجة
- أمر `off` مهم لحماية المحركات من الحرارة والشد

## واجهة التحكم بالمحركات (`web_controller.py` + `templates/index.html`)

### الغرض
واجهة ويب تفاعلية للتحكم بكل المحركات الـ18 (9 يمين + 9 يسار) عبر المتصفح.
تشتغل على البورت **5000** وبتوفر تحكم مباشر بكل سيرفو.

### طريقة التشغيل
```
python3 web_controller.py
```
بعدها افتح المتصفح على: `http://<IP>:5000`

### الميزات
- **Slider لكل محرك** — تحكم بزاوية كل سيرفو من 45° إلى 135°
- **أزرار ±** — تحريك كل محرك درجة واحدة فقط في كل ضغطة
- **القيم الافتراضية** — نفس قيم التوازن من `ground_calibration_test.py`
- **زر تطبيق التوازن** — يرجع كل المحركات لوضعية التوازن
- **زر إيقاف الكل** — يفصل كل السيرفوهات (أمان)
- **حفظ وضعيات** — مثل SharedPreferences، احفظ أي وضعية باسم واسترجعها لاحقاً
- **وضع المحاكاة** — يشتغل حتى بدون هاردوير متصل

### API Endpoints
- `GET /` — صفحة الويب
- `GET /api/status` — حالة كل المحركات
- `POST /api/servo` — تحريك محرك واحد `{key, angle}`
- `POST /api/calibrate` — تطبيق التوازن الافتراضي
- `POST /api/off` — إيقاف الكل
- `GET /api/positions` — قائمة الوضعيات المحفوظة
- `POST /api/positions/save` — حفظ وضعية `{name}`
- `POST /api/positions/load` — تحميل وضعية `{name}`
- `POST /api/positions/delete` — حذف وضعية `{name}`

## ملف ضبط التوازن مع الجايرسكوب (`stance_tune_bno.py`)

### الغرض
ضبط وضعية التوازن للروبوت باستخدام حساس BNO085 (IMU) عبر UART.
بيقيس Roll و Pitch وبيساعد تعدّل زوايا المحركات لتحقيق توازن مثالي.

### الهاردوير الإضافي
- **BNO085 RVC** متصل عبر `/dev/serial0` بسرعة 115200
- بيقرأ: Yaw, Pitch, Roll, Acceleration (ax, ay, az)

### الأوامر التفاعلية
- `apply` — تطبيق كل زوايا التوازن سلس
- `zero` — تصفير مرجع الـ Roll و Pitch (خلي الروبوت ثابت)
- `read` — قراءة Roll/Pitch الحالي (صحيح حسب المرجع)
- `set R/L CH ANGLE` — ضبط زاوية محرك مباشرة
- `step R/L CH DELTA` — زيادة/نقصان زاوية بمقدار محدد
- `status` — عرض كل الزوايا الحالية
- `save` — طباعة القيم للحفظ
- `off` — فصل كل المحركات

### ملاحظات مهمة
- بيستخدم `BNO08x_RVC` (وضع RVC بدون مكتبة I2C)
- قيم المعايرة قريبة من ملف التوازن مع فروق بسيطة (L1=59 بدل 57, L4=76 بدل 74)

## مخطط المشي — Gait Planner

### الغرض
نظام مشي أوتوماتيكي للروبوت العنكبوت. يتيح حفظ خطط المشي (gait plans) وتنفيذها
في thread خلفي باستخدام gait نمط tripod (مثلثي).

### تخزين خطط المشي (`gait_params.json`)
كل خطة تتكون من اسم + معلمات لكل مجموعة أرجل (6 مجموعات: RF, RM, RR, LF, LM, LR).
كل مجموعة تحتوي 4 أوضاع:
- **stance** — وضعية الوقوف
- **lift** — رفع الرجل
- **swing_fwd** — تحريك الرجل للأمام
- **push_back** — دفع الرجل للخلف

### صفحة مخطط المشي
- `GET /gait` — صفحة الويب (`templates/gait.html`)

### Gait API Endpoints
- `GET /api/gait/params` — جلب كل خطط المشي المحفوظة
- `POST /api/gait/params/save` — حفظ خطة `{name, params}`
- `POST /api/gait/params/load` — تحميل خطة `{name}`
- `GET /api/gait/plans` — قائمة أسماء الخطط
- `POST /api/gait/params/delete` — حذف خطة `{name}`
- `POST /api/gait/set-leg-pose` — تطبيق وضعية على رجل واحدة `{group, pose}` (للاختبار)
- `POST /api/gait/auto-start` — بدء المشي `{name, speed, steps}`
  - speed: مضاعف السرعة (0.5=بطيء، 1.0=عادي، 2.0=سريع)
  - steps: 0=غير محدود، N=توقف بعد N خطوة
- `POST /api/gait/auto-stop` — إيقاف المشي فوراً
- `GET /api/gait/status` — حالة المشي `{running, step_count, current_phase}`

### آلية المشي (Tripod Gait)
- **GROUP_A**: RF, LM, LR (أرجل 1,3,5)
- **GROUP_B**: LF, RM, RR (أرجل 2,4,6)
- Phase A: مجموعة A ترفع وتتأرجح للأمام، مجموعة B تدفع للخلف
- Phase B: مجموعة B ترفع وتتأرجح للأمام، مجموعة A تدفع للخلف
- التحريك سلس باستخدام `smooth_move_leg()` مع استيفاء خطي

### ملاحظات مهمة
- المشي يشتغل في background thread (daemon)
- عند الإيقاف، الروبوت يرجع لوضعية stance
- `gait_running` متغير عام للتحكم بإيقاف الـ thread

## مشي العنكبوت المتناغم — Ripple Gait

### الغرض
نوع مشي جديد أهدأ وأثبت من Tripod. رجل واحدة بس ترتفع بالهوا بالتسلسل (مثل موجة من الخلف للأمام) — دايماً 5 أرجل على الأرض.

### الفرق عن Tripod
- Tripod: 3 أرجل بالهوا → الجسم يهتز قليلاً
- Ripple: 1 رجل بالهوا → الجسم ثابت تقريباً، حركة ناعمة متل العنكبوت الحقيقي

### ملف البيانات (`ripple_gait_params.json`)
- `LIFT_DEG`: 14° (رفع خفيف، نص Tripod)
- `SWING_DEG`: 20° (تأرجح Coxa)
- `total_frames`: 60 فريم بدل 8
- التسلسل: RR → RM → RF → LR → LM → LF

### Ripple Gait API Endpoints
- `POST /api/gait/ripple/start` — تشغيل `{speed, cycles, direction}`
  - direction: "forward" أو "backward"
  - cycles: 0=غير محدود، N=توقف بعد N دورة
- `POST /api/gait/ripple/stop` — إيقاف نظيف
- `GET /api/gait/ripple/status` — الحالة `{running, step_count, current_phase}`
